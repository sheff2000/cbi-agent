Директория конфигурации Агента

Файлы и параметры модулей будут наполнятся по мере написания и расштирения Агента

xml/ - директория с кастомными значениями параметров Агента. Значения устанавливаются на 
        фронте админом или владельцем(цами) Агента по ws-control
        Файлы могут быть пустыми (или отсутствовать) если используются все значения по умолчанию
        Может содержать только часть значений которые были изменены.

modeules/ - содержит константы/обьекты и глобальные переменные 
            для конфигуарции нужного модуля агента (коннект, RC, video, metrica...)
            Содержит значения по умолчанию.
            Если есть значение в соответствующем xml - то используется оно

utils/ - набор утилит для организации конфига (тот же парсер xml)

src/
├──config/
    ├── xml/ #файлы могут отсутствовать / появляются если задаются конфиги с фронта
    │   ├── connect.xml    # точка подключения, приоритеты серверов, SSL, etc.
    │   ├── video.xml      # параметры кодека, битрейт, MTU, overlay
    │   ├── rc.xml         # RC/serial параметры, deadzone, частоты
    │   └── system.xml     # общие параметры (например, hardware_id, флаги отладки)
    │
    ├── modules/ # все параметры конфигураций по умолчанию либо значения с соответствующего XML
    │   ├── connect.js
    │   ├── video.js
    │   ├── rc.js
    │   └── telemetry.js
    └── utils/
        └── xmlLoader.js   # универсальный загрузчик и парсер XML


В каждом модуле при инициализации загружаем возможный .xml с кастомными параметрами
import { loadConfig } from '../utils/xmlLoader.js';
const cfg = loadConfig('video');
const params = {
    codec: cfg.codec || 'h264-pass',
    bitrate: cfg.bitrate || 1200,
    width: cfg.width || 720,
    height: cfg.height || 576,
};
